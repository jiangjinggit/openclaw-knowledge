# 门禁检查清单

## 使用说明

在发布内容或产品前，使用此清单进行质量检查。

---

## 内容门禁（文章/文档）

### 基础检查 ✅

- [ ] **无错别字**
  - 使用工具检查（如 Word、Grammarly）
  - 人工复查一遍
  - 特别注意专有名词

- [ ] **无敏感词**
  - 政治敏感词
  - 违禁词
  - 侵权内容

- [ ] **格式正确**
  - 标题层级正确（H1/H2/H3）
  - 列表格式统一
  - 代码块格式正确
  - 表格显示正常

- [ ] **链接有效**
  - 所有链接可以打开
  - 没有死链
  - 跳转正确

### 内容质量 ✅

- [ ] **有数据支撑**
  - 关键观点有数据
  - 数据来源可靠
  - 数据标注清晰（来源/时间）

- [ ] **有配图**（如需要）
  - 配图清晰
  - 配图相关
  - 配图有版权或自制

- [ ] **结构清晰**
  - 有明确的开篇
  - 有清晰的章节
  - 有明确的结论

- [ ] **可读性好**
  - 段落不超过5行
  - 句子不超过30字
  - 避免专业术语（或有解释）

### 发布准备 ✅

- [ ] **文件已备份**
  - 本地备份
  - 云端备份
  - 版本控制（Git）

- [ ] **元数据完整**
  - 标题
  - 摘要
  - 标签
  - 封面图

- [ ] **发布渠道确认**
  - 公众号
  - 小红书
  - 知乎
  - 其他平台

---

## 代码门禁（功能/产品）

### 基础检查 ✅

- [ ] **测试通过**
  - 单元测试通过
  - 集成测试通过
  - 测试覆盖率达标（建议 >80%）

- [ ] **代码已 review**
  - 至少1人 review
  - 所有评论已解决
  - 符合代码规范

- [ ] **有技术文档**
  - API 文档
  - 使用示例
  - 故障排查指南

- [ ] **无明显 bug**
  - 核心功能正常
  - 边界情况处理
  - 错误提示友好

### 安全检查 ✅

- [ ] **无安全漏洞**
  - SQL 注入
  - XSS 攻击
  - CSRF 攻击
  - 敏感信息泄露

- [ ] **权限控制正确**
  - 认证机制
  - 授权检查
  - 数据隔离

- [ ] **依赖安全**
  - 使用 `npm audit` 检查
  - 无高危漏洞
  - 依赖版本合理

### 性能检查 ✅

- [ ] **性能达标**
  - 响应时间 <100ms（API）
  - 页面加载 <3s
  - 无内存泄漏

- [ ] **资源优化**
  - 图片压缩
  - 代码压缩
  - 缓存策略

- [ ] **可扩展性**
  - 支持并发
  - 数据库索引
  - 缓存机制

### 发布准备 ✅

- [ ] **文件已备份**
  - 代码已提交
  - 数据库已备份
  - 配置文件已备份

- [ ] **回滚预案**
  - 回滚脚本准备
  - 回滚步骤文档
  - 回滚测试通过

- [ ] **监控就绪**
  - 日志记录
  - 错误监控
  - 性能监控

## 数据门禁（报告/分析）

### 基础检查 ✅

- [ ] **数据准确**
  - 数据来源可靠
  - 计算方法正确
  - 无明显错误

- [ ] **数据完整**
  - 样本量足够
  - 时间范围明确
  - 维度覆盖全面

- [ ] **数据可视化**
  - 图表清晰
  - 标签完整
  - 颜色区分明显

### 分析质量 ✅

- [ ] **结论明确**
  - 有明确的结论
  - 结论有数据支撑
  - 结论可执行

- [ ] **建议具体**
  - 有具体的建议
  - 建议有优先级
  - 建议有 owner 和 deadline

- [ ] **风险提示**
  - 数据局限性
  - 分析假设
  - 不确定性

### 发布准备 ✅

- [ ] **文件已备份**
  - 原始数据
  - 分析脚本
  - 报告文档

- [ ] **可复现**
  - 数据来源记录
  - 分析步骤记录
  - 工具版本记录

## 门禁失败处理

### 如果门禁失败

1. **记录失败原因**
   - 哪个检查项失败
   - 具体错误信息
   - 失败时间

2. **修复问题**
   - 根据错误信息修复
   - 重新运行检查
   - 确认修复成功

3. **更新状态**
   - 更新任务状态
   - 通知相关人员
   - 记录到 MEMORY.md

4. **重新检查**
   - 运行完整门禁检查
   - 确认所有项通过
   - 继续发布流程

## 门禁豁免（慎用）

### 何时可以豁免

1. **紧急修复**
   - 线上严重 bug
   - 安全漏洞修复
   - 数据丢失风险

2. **实验性功能**
   - 内部测试
   - A/B 测试
   - 灰度发布

3. **临时方案**
   - 快速验证
   - 原型演示
   - 概念验证

### 豁免流程

1. **申请豁免**
   - 说明豁免原因
   - 评估风险
   - 获得 CEO 批准

2. **记录豁免**
   - 记录到 `team/DECISIONS.md`
   - 说明豁免原因
   - 说明风险和影响

3. **后续补救**
   - 设置补救 deadline
   - 分配 owner
   - 跟踪完成情况

## 常见问题

### Q: 门禁检查需要多长时间？
A: 
- 内容门禁：5-10分钟
- 代码门禁：10-20分钟
- 数据门禁：5-10分钟

### Q: 门禁检查谁来做？
A: 
- 内容门禁：Build + Ops
- 代码门禁：Build + Ops
- 数据门禁：Research + Ops

### Q: 门禁失败怎么办？
A: 
1. 记录失败原因
2. 修复问题
3. 重新检查
4. 确认通过后继续

### Q: 可以跳过门禁吗？
A: 
- 原则上不可以
- 紧急情况可以豁免
- 必须获得 CEO 批准
- 必须记录到 DECISIONS.md
