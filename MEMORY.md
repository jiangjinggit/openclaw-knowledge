# MEMORY.md - 长期记忆模板

## 使用说明

这个文件用于记录跨会话的长期记忆，包括：
- 每次犯错的教训
- 每次学习的经验
- 每次重要的决策

**记录格式：**
```markdown
### [日期] [类型] [标题]
- **问题**：[发生了什么]
- **根因**：[为什么会发生]
- **修正**：[怎么解决的]
- **预防**：[下次怎么避免]
```

**类型：**
- `ERROR`：错误/失败
- `LEARN`：学习/经验
- `DECISION`：重要决策

## 记忆记录

### 2026-03-06 ERROR 示例：任务没有验收标准
- **问题**：Build 完成任务后，我觉得不满意，但说不出哪里不满意
- **根因**：任务卡没有明确的验收标准
- **修正**：重新定义任务，增加验收标准："3000字、3个真实案例、通过门禁检查"
- **预防**：以后每个任务都必须有可量化、可验证的验收标准

### 2026-03-06 LEARN 示例：配置文件要简洁
- **问题**：第一版 SOUL.md 写了 50 条规则，AI 执行混乱
- **根因**：规则太多，AI 无法判断优先级
- **修正**：简化为 10 条核心规则
- **预防**：从简单开始，逐步优化；每次只加 1-2 条规则

### 2026-03-06 DECISION 示例：选择 OpenClaw
- **问题**：需要一个 AI 协作系统
- **选择**：OpenClaw（而不是 ChatGPT、Claude、Cursor）
- **原因**：
  1. 主动执行（定时任务）
  2. 团队协作（多 Agent）
  3. 完全本地（数据安全）
- **结果**：效率提升 8 倍

## 使用建议

### 什么时候记录？
1. **犯错时**：立即记录，避免下次重复
2. **学到新东西时**：记录下来，方便以后查阅
3. **做重大决策时**：记录原因和结果，方便复盘

### 怎么记录？
1. **简洁**：一条记录不超过 100 字
2. **具体**：不要写"注意细节"，要写"检查验收标准"
3. **可执行**：预防措施要可执行，不要写"下次小心"

### 什么时候查阅？
1. **遇到类似问题时**：先查 MEMORY.md，看有没有记录
2. **开始新任务前**：看看有没有相关经验
3. **定期复盘时**：每周/每月回顾一次

### 什么时候清理？
1. **过时的记录**：如果某条规则已经固化到配置文件，可以删除
2. **重复的记录**：如果多条记录说的是同一件事，合并成一条
3. **无用的记录**：如果某条记录从来没用过，可以删除

## 记忆分类

### 错误类（ERROR）
记录失败和教训，避免重复犯错。

**示例：**
```markdown
### 2026-03-06 ERROR 数据丢失
- **问题**：删除文件时没有备份，数据丢失
- **根因**：没有养成备份习惯
- **修正**：从备份恢复（幸好有自动备份）
- **预防**：删除前必须先备份；设置自动备份
```

### 学习类（LEARN）
记录学到的经验和最佳实践。

**示例：**
```markdown
### 2026-03-06 LEARN 标题公式
- **问题**：文章阅读量低
- **学习**：好标题 = 趋势词 + 冲突/对比 + 数据锚点
- **应用**：下次写标题时使用这个公式
- **结果**：阅读量提升 3 倍
```

### 决策类（DECISION）
记录重要决策的原因和结果，方便复盘。

**示例：**
```markdown
### 2026-03-06 DECISION 选择技术栈
- **问题**：需要选择前端框架
- **选择**：React（而不是 Vue、Angular）
- **原因**：
  1. 生态最好
  2. 招人容易
  3. 我最熟悉
- **结果**：开发速度快，没有踩坑
```

## 常见问题

### Q: MEMORY.md 和 SOUL.md 有什么区别？
A: 
- SOUL.md 是"价值观"，定义了 AI 的行为准则（不常变）
- MEMORY.md 是"记忆"，记录了具体的经验教训（经常变）

### Q: 什么时候把 MEMORY.md 的内容写入 SOUL.md？
A: 
当某条经验教训被验证有效，并且需要成为"行为准则"时，可以写入 SOUL.md。

**示例：**
```
MEMORY.md: "2026-03-06 ERROR 任务没有验收标准"
↓ 验证有效后
SOUL.md: "任务必须具备 owner、deadline、acceptance"
```

### Q: MEMORY.md 会不会越来越长？
A: 
会的。建议：
1. 每月清理一次，删除过时/重复/无用的记录
2. 把重要的记录固化到 SOUL.md 或 AGENTS.md
3. 把详细的记录归档到 `memory/archive/` 目录

### Q: 可以不用 MEMORY.md 吗？
A: 
可以，但不推荐。MEMORY.md 是 AI 自我进化的关键，没有它，AI 会重复犯错。
