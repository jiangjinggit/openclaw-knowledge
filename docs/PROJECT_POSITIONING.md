# OpenClaw Knowledge 项目定位

## 一句话定位
`openclaw-knowledge` 是 OpenClaw 生态里的 **知识沉淀 / 教学模板 / 方法论资产仓库**。

## 当前角色
当前仓库不是工具型产品，不承担：
- 运行监控
- 安全审计
- 商业机会筛选
- 模板商店展示

它当前承担的是：
- 多 Agent 协作配置模板
- 新手上手与安装说明
- 协作规则与任务/结果模板
- 飞书知识库版教学材料

## 未来扩展角色
### 1. 内容变现底座
- 公众号内容的结构母版
- 小红书/知乎/公众号教程骨架
- 检查清单、SOP、模板包

### 2. 培训交付底座
- 训练营讲义
- 顾问交付材料
- 企业内训模板
- 演示环境模板

### 3. 方法论沉淀底座
- 多 Agent 协作方法论
- 角色拆分原则
- 门禁与验收规则
- 从个人使用到团队协作的迁移路径

## 与其他仓库的边界
### `openclaw-money-radar`
回答：**什么值得做、怎么赚钱**

### `openclaw-template-market`
回答：**什么模板值得卖、怎么展示和成交**

### `openclaw-monitor-lite`
回答：**OpenClaw 怎么监控、告警、看成本和运行状态**

### `openclaw-security-audit-lite`
回答：**OpenClaw 是否安全、如何体检、如何整改**

### `openclaw-knowledge`
回答：**如何把 OpenClaw 的经验沉淀成可学习、可复用、可教学的知识资产**

## 当前最适合做的事
1. 把多 Agent 协作模板继续打磨成更标准的知识产品
2. 增加“适合谁 / 怎么用 / 能得到什么”的对外表达
3. 逐步补内容资产分类：教程、模板、训练营、SOP、FAQ
4. 与公众号/小红书内容联动，沉淀成长期知识库
