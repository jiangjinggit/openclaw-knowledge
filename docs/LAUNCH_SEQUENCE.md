# OpenClaw Knowledge 上线顺序建议

## 目标
明确 `openclaw-knowledge` 不应该一次性把所有内容产品都推出，而应该按更容易成交、更容易验证的顺序逐步上线。

---

## Phase 1：打基础认知
### 目标
先让仓库具备“能看懂、能导航、能理解定位”的公开展示基础。

### 当前已完成
- README 定位
- 项目边界说明
- 资产地图
- 免费到付费路径
- 产品分层设计
- 渠道策略
- lead magnet 清单

### 成果
- 仓库已经具备对外展示与后续承接能力

---

## Phase 2：推出第一个低价产品
### 推荐优先级 1
**多 Agent 协作模板包**

### 原因
- 和当前仓库内容最贴近
- 打包成本最低
- 最容易先成交
- 最适合从 GitHub 和内容流量承接

### 配套动作
- 做产品说明页
- 做简版 lead magnet
- 在公众号/小红书发内容导流

---

## Phase 3：推出第二个更强结构化产品
### 推荐优先级 2
**AI 公司配置模板包**

### 原因
- 比基础模板包更有产品感
- 更适合承接高意向用户
- 更像你个人方法论的商品化表达

### 配套动作
- 增加角色说明与场景示例
- 形成前后版本差异说明
- 作为训练营前置产品

---

## Phase 4：推出交付型模板产品
### 推荐优先级 3
**飞书知识库交付模板包**

### 原因
- 面向更高意向和更高客单价场景
- 更适合顾问、培训、团队协作用户
- 能拉开与普通模板产品的差异

---

## Phase 5：推出训练营 / 工作坊升级产品
### 推荐优先级 4
**训练营资料包 / 工作坊版本**

### 原因
- 当你已有模板、方法论、渠道内容后，训练营才更容易成立
- 客单更高，但前提是前面三层先跑通

---

## 不建议的顺序
### 不建议一开始就做
- 大而全课程体系
- 企业内训重交付版本
- 过多模板同时上线
- 太多平台同时分发

### 原因
- 会稀释仓库定位
- 验证速度慢
- 容易失去“先成交一个小产品”的节奏

---

## 当前最推荐的下一步
1. 继续保持仓库结构清晰
2. 优先准备“多 Agent 协作模板包”的对外说明
3. 配一份免费的简版清单做引流
4. 再逐步升级到 AI 公司配置模板包与飞书模板包
