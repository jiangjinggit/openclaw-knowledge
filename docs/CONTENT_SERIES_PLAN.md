# OpenClaw Knowledge 内容系列规划

## 目标
围绕 `openclaw-knowledge` 持续产出内容，不是零散写文章，而是形成稳定系列。

---

## 系列 1：OpenClaw 多 Agent 入门系列
### 适合平台
- 小红书
- 公众号
- GitHub 导航页外链

### 选题方向
- 多 Agent 协作是什么
- 最小可用配置怎么搭
- 5 个核心文件分别负责什么
- 新手最容易犯的配置错误

### 目标
- 吸引新手
- 承接基础模板包

---

## 系列 2：AI 公司配置模板系列
### 适合平台
- 公众号
- 知乎
- 私域长文

### 选题方向
- CEO / PM / Build / Research / Growth / Ops 如何分工
- 为什么多 Agent 协作容易失控
- 怎么把个人工作流升级成“AI 公司”结构
- 角色模板如何搭配任务卡和结果卡

### 目标
- 承接 AI 公司配置模板包
- 建立方法论心智

---

---

## 推荐发布顺序
1. 多 Agent 入门系列
2. AI 公司配置模板系列
3. 内容工厂与知识产品系列
4. 飞书交付与培训系列

---

## 执行原则
1. 每个系列都要服务一个明确产品。
2. 每篇内容都要有承接动作：模板、清单、表单或升级产品。
3. 不做无承接的纯观点输出。
