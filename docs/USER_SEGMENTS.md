# OpenClaw Knowledge 用户分层

## 目标
明确 `openclaw-knowledge` 面向的不是同一种用户，而是不同阶段、不同付费意愿、不同需求深度的人群。

---

## 第一层：新手探索用户
### 特征
- 刚接触 OpenClaw
- 还在理解多 Agent 协作是什么
- 更关注“怎么开始”而不是“怎么优化”

### 最关心的问题
- 我该先看什么？
- 这些配置文件到底是干嘛的？
- 我能不能先跑起来一个最小版本？

### 适合承接的内容
- README
- INSTALL / USAGE
- 导航文档
- 新手路线图
- 基础免费清单

### 最适合的产品
- 免费模板
- 新手清单
- 基础模板包

---

## 第二层：模板购买用户
### 特征
- 已经有明确用途
- 不想从零整理模板
- 愿意花小钱换时间

### 最关心的问题
- 有没有现成模板可以直接用？
- 哪个模板最适合我的场景？
- 我买了之后能不能马上改起来？

### 适合承接的内容
- 模板包说明
- 模板差异说明
- 任务卡/结果卡/SOP 示例

### 最适合的产品
- 多 Agent 协作模板包
- AI 公司配置模板包
- SOP 包

---

## 第三层：内容创作者 / 方法论用户
### 特征
- 运营公众号、小红书、知乎等内容渠道
- 不只想搭配置，还想把经验沉淀成内容资产
- 对方法论和知识结构感兴趣

### 最关心的问题
- 我如何把这些模板转成内容？
- 怎么做清单、教程、lead magnet？
- 怎么把内容和产品结合起来？

### 适合承接的内容
- 内容产品计划
- Lead magnets 清单
- 渠道策略
- 方法论文档

### 最适合的产品
- 内容模板包
- lead magnet 包
- 方法论包

---

## 第四层：培训 / 顾问 / 交付型用户
### 特征
- 想把这套东西用于教学、训练营、顾问交付
- 更重视知识库结构、交付模板、飞书版材料

### 最关心的问题
- 我能不能把这套东西拿去交付给客户/学员？
- 有没有飞书版、培训版、讲义版？
- 如何提升我的交付效率？

### 适合承接的内容
- 飞书模板
- 训练营资料规划
- 企业内训模板
- 顾问交付资料

### 最适合的产品
- 飞书知识库模板包
- 训练营资料包
- 企业内训模板包

---

## 第五层：团队 / 企业升级用户
### 特征
- 已不满足于单人学习或模板套用
- 关心团队协作、角色分工、知识沉淀和交付规范

### 最关心的问题
- 如何给团队建立一套统一协作模板？
- 如何把知识和模板组织成内部资产？
- 如何用于企业试点或培训？

### 适合承接的内容
- 企业交付模板
- 角色设计原则
- 团队协作方法论
- 飞书知识库结构

### 最适合的产品
- 企业内训版
- 顾问升级包
- 定制交付模板包

---

## 使用建议
每次新增内容或设计产品时，优先明确：
1. 它服务哪一层用户？
2. 它是帮助用户入门、成交、升级，还是提高客单？

如果两点都不清楚，就不建议直接塞进主仓库。
