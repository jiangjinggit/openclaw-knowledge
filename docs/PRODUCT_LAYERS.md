# OpenClaw Knowledge 产品分层设计

## 目标
把 `openclaw-knowledge` 从“知识仓库”进一步升级为：

> **可同时服务免费用户、低价知识产品用户、培训/咨询升级用户的内容与方法论产品底座**

---

## 第一层：免费开源层
### 目标
建立认知、获取搜索流量、降低首次上手门槛。

### 适合承载的内容
- README 与快速开始
- 基础配置模板
- 安装说明
- 多 Agent 协作基础规则
- FAQ / 常见误区
- 飞书知识库基础版

### 用户获得什么
- 知道 OpenClaw 多 Agent 协作的基本结构
- 能完成第一版配置
- 能理解项目方法论框架

### 作用
- 认知入口
- GitHub 流量入口
- 内容引流落点

---

## 第二层：低价知识产品层
### 目标
把公开内容转成可快速成交的下载型产品。

### 适合承载的产品
- 多 Agent 配置模板包
- AI 公司协作模板包
- 任务卡 / 结果卡 / 门禁清单包
- 飞书知识库模板包
- SOP / 检查清单 / 教学配套文档

### 用户获得什么
- 拿来即用的模板
- 更完整的协作结构
- 更标准的交付材料
- 节省自己从零搭框架的时间

### 建议价格区间
- `19-99 元`：单个模板/清单
- `99-299 元`：模板包 / 方法论包
- `299-699 元`：系统化知识模板包

---

## 第三层：培训与咨询升级层
### 目标
把模板与方法论升级为更高客单的培训交付产品。

### 适合承载的产品
- 训练营讲义
- 企业内训模板
- 顾问交付前置材料
- 多 Agent 协作工作坊材料
- 团队协作设计模板

### 用户获得什么
- 系统化理解
- 定制化场景应用
- 团队协作设计能力
- 交付能力提升

### 建议价格区间
- `299-999 元`：小课 / 训练营资料
- `999-3999 元`：训练营 / 工作坊
- `5000 元+`：企业内训 / 顾问交付版

---

## 第四层：长期方法论资产层
### 目标
沉淀长期复利资产，而不只是卖一次性模板。

### 适合承载的内容
- 多 Agent 协作原则
- 不同角色分工范式
- 内容工厂 / 产品工厂 / 研发协作方法论
- 从个人使用到团队协作的迁移框架
- 内容与模板联动策略

### 作用
- 支撑公众号/小红书长期内容
- 支撑培训产品升级
- 支撑顾问和企业版高客单
- 支撑未来拆分更多子产品

---

## 仓库经营原则
1. 免费层负责引流，不负责承载全部价值。
2. 低价产品层负责快速成交。
3. 培训升级层负责提高客单。
4. 方法论资产层负责长期复利。
5. 每次新增内容时，都优先判断它属于哪一层，不要无序堆积。
